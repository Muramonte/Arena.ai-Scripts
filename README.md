## chat export
###1.2
⚠️ Оговорка: заголовки завязаны на конкретные Tailwind-классы (`justify-end`, `bg-surface-primary`, `span.truncate`), которые сайт может поменять при обновлении интерфейса — тогда нумерация просто перестанет срабатывать (но сам текст всё равно соберётся через fallback).

**Что отличает пользователя от модели в HTML:**

**Пользователь** — контейнер-обёртка содержит класс с `justify-end`:
```html
<div class="mx-auto max-w-[800px] flex w-full justify-end">
```

**Модель** — контейнер-обёртка имеет класс `bg-surface-primary` и содержит имя модели в `<span class="truncate">gpt-5.2-chat-latest</span>` внутри шапки.

Оба типа сообщений оборачивают текст в `<div class="prose ...">` — это удобная точка входа: беру весь `.prose`-блок целиком как **одно сообщение** (а не бьётся на отдельные `<p>`/`<li>` как раньше), а роль (user/assistant + имя модели) определяю, поднимаясь по родителям вверх от этого блока.

**Изменения:**

- Новая функция `collectMessageBlocks` берёт целиком каждый блок `.prose` как одно сообщение (не дробит на отдельные абзацы/списки, как раньше).
- `detectRole` поднимается по родителям от `.prose`-блока и определяет: класс с `justify-end` → пользователь, класс `bg-surface-primary` (+ имя модели из `span.truncate`) → ассистент.
- `assignNumbers` нумерует: каждое сообщение пользователя увеличивает главный счётчик (`#1`, `#2`...), каждый ответ после него — подсчётчик (`#1.1`, `#1.2`...), который сбрасывается при новом сообщении пользователя. Это уже готово к side-by-side — если после одного `#1` будет два подряд ответа модели, они получат `#1.1` и `#1.2`.
- В markdown перед каждым сообщением добавляется заголовок вида `### #1.1 gpt-5.2-chat-latest` или `### #1 Вы`.
- Если структура `.prose`/классов на странице не находится (например, старый lmsys-сайт без такой вёрстки) — скрипт автоматически откатывается на прежнюю генерику без нумерации, а затем на полный текст страницы. Ничего не должно сломаться на других сайтах из `@match`.

Пример вывода:
```
### #1 Вы

1

### #1.1 gpt-5.2-chat-latest

2
```

### 1.1
- кнопка внизу по центру "Export"
- поддержка заголовков `h1`–`h6` (включая короткие, вроде `#1.1`)
- заголовки не режутся фильтром минимальной длины
- заголовки сериализуются в markdown (`#`, `##`, ...)
- исправлено дублирование текста (контейнер + вложенные блоки) — теперь после сбора отбрасываются элементы-контейнеры, если внутри них уже есть отдельно собранные блоки
- нумерованные списки (`<ol>`) сохраняют номера, а не превращаются в `-`
- пустые ссылки (например, иконка без текста) больше не теряют `href`
- убран мёртвый код (`sleep()`, неиспользуемая константа `HOTKEY`)
- добавлен стабильный порядок сортировки (индекс при сборе) как дополнительный тай-брейкер, чтобы при почти одинаковых координатах порядок не был случайным
  
- стоит `@match` только для arena.ai. Можно поставить `// @match        *://*/*` чтобы скрипт работал на любом сайте.
